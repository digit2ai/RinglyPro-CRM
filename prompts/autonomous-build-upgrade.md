# Autonomous Build â€” Troubleshooting Guide

## Quick status check
- Render dashboard â†’ service `aiagent-ringlypro-com` â†’ Logs tab
- Filter logs for `[autonomousBuild]` or `[architectPipeline]`
- Project state in DB:

  SELECT id, name, workflow_phase, build_status, short_name,
         build_iterations, build_started_at
  FROM d2_projects
  WHERE workflow_phase IN ('queued','manual_build','build_stuck')
  ORDER BY build_started_at DESC LIMIT 20;

## Feature flag
- AUTONOMOUS_BUILD_ENABLED=false  â†’ old email-to-Manuel handoff
- AUTONOMOUS_BUILD_ENABLED=true   â†’ autonomous build active
  (set in Render â†’ Environment â†’ save â†’ wait for service restart ~30s)

## Common failure modes

### "Build started" email never arrives
- Check Render logs for `[autonomousBuild] config check` line
- Verify env vars on Render: ANTHROPIC_API_KEY, AUTONOMOUS_BUILD_ENABLED,
  BUILD_GIT_TOKEN
- Check SendGrid logs at https://app.sendgrid.com/email_activity
- The build still runs even if the email fails (send failure is logged
  but does not abort)

### Project stuck at workflow_phase='queued'
- Means the architect prompt was saved but the agent never picked it up
- Hourly safety-net poller (autonomousBuild.startPoller) catches stale
  queued projects after 5 minutes; check Render logs for it
- Manual unstick: SET workflow_phase='build_authorized', run
  POST /projects/api/v1/projects/:id/set-phase via the admin Override
  Phase button on the project detail page

### Agent ran but project is in 'build_stuck'
- Open the SIT-failure email â€” failure_stage tells you which step
- workspace_path in the email points to /tmp/build-<short_name>-<ts>/ on
  the Render instance. Render filesystem is ephemeral, so this workspace
  is GONE once the service restarts. Inspect via Render Shell immediately
  if needed.
- Common causes:
  - agent_init: BUILD_GIT_TOKEN missing or invalid; permission denied on
    clone
  - agent_run: model rejected the prompt (too long, content filter)
  - token_budget: prompt too complex; bump BUILD_MAX_TOKENS temporarily
  - lint_repair_exhausted: agent wrote code that won't parse; bump
    BUILD_MAX_TURNS or rewrite the prompt before retrying
  - git_push: branch protection or stale ref; check repo settings

### Build pushed but Render is not deploying
- Confirm the commit landed: git log origin/main -- client-builds/<short_name>/
- Check Render's deploy history; the auto-deploy hook may be paused
- Manually trigger a deploy from Render dashboard

### Build pushed, deployed, but SIT failed
- This is the existing pipeline doing its job, not an autonomous build
  problem. Check the SIT-failure email (existing flow), inspect
  ${production_url}/health manually, re-build by editing the code in
  client-builds/<short_name>/ and pushing.

### Want to disable autonomous build entirely (emergency stop)
- Render â†’ Environment â†’ set AUTONOMOUS_BUILD_ENABLED=false â†’ save
- Save triggers a 30-second restart; all in-flight agents will be killed
  (Render kills the process)
- New projects entering build_authorized will fall back to the old
  email-to-Manuel handoff
- In-flight projects already in 'queued' may be left stranded; mark them
  build_authorized manually

### Cancel an in-flight build for one project
- Click the "Stop this build" link in the build-started email
- Or POST /projects/api/v1/projects/:id/cancel-autonomous-build with
  admin auth
- The agent checks for a cancel flag every 30s and exits cleanly;
  project goes back to build_authorized

## Cost monitoring
- Each build run logs tokens_used to d2_activity_log with action=
  'autonomous_build_completed' or 'autonomous_build_failed'
- Aggregate spend:

  SELECT SUM((details->>'tokensUsed')::int) AS tokens
  FROM d2_activity_log
  WHERE action LIKE 'autonomous_build_%'
    AND created_at > NOW() - INTERVAL '30 days';

- Anthropic dashboard: https://console.anthropic.com/settings/usage
- Hard ceiling per build: BUILD_MAX_TOKENS env var (default 500000)

## When to roll back to manual handoff
- Two consecutive autonomous builds fail on the same kind of error
- Token spend in a 24h window exceeds your monthly budget pace
- Agent produces code that passes SIT but fails UAT review repeatedly
  (suggests prompt quality, not infrastructure)

Set AUTONOMOUS_BUILD_ENABLED=false, hold for analysis, fix prompts or
infrastructure, re-enable on a single test project before broad rollout.
