‚ö†Ô∏è Key Differences: Current vs Future State

Current State (What Exists Now)

  • Language-based routing (Rachel EN / Ana ES)
  • No health check or failover logic
  • HTTP API proxies (custom implementation)
  • ElevenLabs TTS only (no Conversational Agent)
  • Sequential webhook flow (Twilio TwiML)
  • Polly as TTS fallback only
  • 3 separate Lina implementations (legacy)
  • No circuit breaker pattern

Future State (Planned)

  • ElevenLabs Conversational Agent as PRIMARY
  • Health check + automatic failover to Rachel
  • True MCP Protocol server
  • MCP tools (search_contacts, book_appointment, etc.)
  • ElevenLabs MCP SSE connection
  • Rachel as FALLBACK (not primary)
  • Unified voice routing with health monitoring
  • Centralized tool execution via MCP
