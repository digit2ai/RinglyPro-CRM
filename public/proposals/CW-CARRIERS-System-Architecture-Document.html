1 System Overview & Purpose

What CW Carriers does and why it exists

CW Carriers is a full-stack AI freight brokerage platform that enables CW Carriers USA to:

  1. Manage the complete freight lifecycle — from shipper quotes to carrier dispatch, check calls, delivery confirmation, and dual-sided billing
  2. Automate carrier outreach via Voice AI — Rachel, an ElevenLabs-powered AI agent, handles inbound/outbound calls for load coverage, check calls, and carrier qualification
  3. Price loads with AI-powered rate intelligence — historical data, corridor analysis, and market benchmarks (DAT/TruckStop/GreenScreens) drive confident quoting
  4. Match loads to carriers automatically — multi-factor scoring engine considers lane history, equipment, capacity, reliability, and deadhead optimization
  5. Sync bidirectionally with HubSpot CRM — contacts, deals, and call logs flow between platforms with full audit trail
  6. Provide natural language command interface — Claude AI-powered NLP lets dispatchers manage operations via conversational queries

Layer 1 — Data Input & Ingestion
Multi-channel data ingestion: web forms, CSV/Excel bulk upload (loads, carriers, rates), McLeod TMS bi-directional sync, HubSpot webhook events. Auto-column mapping with alias engine for CSV flexibility.
Layer 2 — RinglyPro MCP Server
Node.js/Express application with 22 route modules, JWT + RBAC authentication, NLP command processor (Claude AI), event router, and PostgreSQL storage (23 tables across cw_* and lg_* prefixes).
Layer 3 — AI Intelligence Engine
Rate Intelligence (historical + benchmark + corridor analysis), Carrier Matching (multi-factor scoring), Load-to-Load Pairing (backhaul/chain optimization), FMCSA Compliance Checker, Brokerage Analytics, NLP Command Processing, Alert Engine.
Layer 4 — Voice AI Engine (Rachel)
ElevenLabs WebRTC voice synthesis + Twilio SIP/PSTN for inbound/outbound calls. Rachel handles check calls, carrier coverage, load quoting, carrier qualification. Transcripts auto-summarized and logged to HubSpot.
Layer 5 — Output & Delivery
React + Vite SPA dashboard, 22-module REST API (JSON), PDF/CSV report generation, HubSpot bi-directional CRM sync with full audit trail.
Layer 6 — External Systems
HubSpot CRM (contacts, deals, activities), McLeod TMS (order/status sync), FMCSA SAFER API (authority/insurance), DAT/TruckStop/GreenScreens (market rates), Twilio (voice/SMS), ElevenLabs (AI voice).

5 Voice AI Engine (Rachel)

ElevenLabs-powered conversational AI agent for freight brokerage operations

Rachel is a production voice AI agent purpose-built for freight brokerage. She handles both inbound and outbound calls using ElevenLabs WebRTC for real-time voice synthesis and Twilio for telephony infrastructure.

Outbound Carrier Coverage

Automated Dispatch

Rachel calls carriers from the matched carrier list to secure load coverage. Negotiates rates, confirms equipment availability, and books carriers with rate confirmations.

Check Calls

Transit Monitoring

Automated check calls to drivers and carriers during transit. Captures location, ETA updates, and exceptions. Logs all results to cw_check_calls with GPS data.

Inbound Handling

Call Routing & Qualification

Handles inbound carrier and shipper calls. Qualifies new carriers (MC/DOT, equipment, lanes), takes load inquiries, and escalates to live dispatch when needed.

Transcript Processing

AI Summary & CRM Sync

Every call is transcribed, summarized by Claude AI, and automatically logged to HubSpot as a call activity. Structured data extracted for load/carrier records.

7 AI Load & Carrier Matching

Multi-factor scoring engine for optimal carrier-to-load and load-to-load pairing

Carrier-to-Load Matching

When a load is posted, the matching engine scores all eligible carriers using a weighted multi-factor algorithm:

FactorWeightScoring Logic
Lane History25%Past loads on same or nearby lanes. Exact city match = 100, same state = 60, adjacent state = 30.
Equipment Match20%Carrier's registered equipment types vs. load requirement. Exact = 100, compatible = 70.
Reliability Score20%Composite of on-time %, acceptance rate, and claims history. Range 0-100.
Rate Fit15%Carrier's preferred rate vs. target buy rate. Within 5% = 100, within 15% = 70, >15% = 30.
Proximity10%Deadhead miles from carrier's current/home location to pickup. <50mi = 100, <150mi = 70, <300mi = 40.
Recency10%Days since last completed load. Active carriers scored higher. <7 days = 100, <30 = 70, <90 = 40.

Load-to-Load Pairing

The pairing engine finds profitable multi-load combinations for carriers:

Backhaul

Return Trip Match

Load B's origin near Load A's destination. Minimizes empty miles on return. Scored by route alignment, timing window, and equipment compatibility.

Chain

Multi-Stop Sequence

Sequential loads forming a continuous route. Optimized for total RPM across the chain with minimal deadhead between stops.

Round Trip

Full Circuit

Origin → Destination → Origin. Both legs covered with freight. Highest utilization score when timing aligns.

Relay

Split Long Haul

Long-haul loads split into segments with carrier handoff at relay points. Enables regional carriers on long lanes.
