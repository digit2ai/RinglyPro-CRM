Volume Projection Engine

Converting Surgical Volume into Robotic Case Forecasts

The foundation of every analysis. Takes total annual surgical volume and specialty mix, then projects how many cases are convertible to robotic surgery using evidence-based adoption rates.

Specialty-Weighted Conversion Core Formula

Each surgical specialty has a different robotic adoption ceiling based on clinical evidence and market data.

// Conversion rates by specialty (from clinical adoption data)
Urology = 85% ← Prostatectomy is near-universally robotic
Gynecology = 45% ← Growing but contested (JAMA 2019)
Colorectal = 35% ← Growing especially rectal cancer
Thoracic = 30% ← Lobectomy/mediastinal
General = 25% ← Hernia, cholecystectomy
Head & Neck = 20% ← Transoral — niche
Cardiac = 15% ← Still early stage

// Per-specialty convertible volume
Convertible[spec] = TotalSurgical x (SpecialtyPct / 100) x ConversionRate[spec]

// Weighted overall conversion rate
WeightedRate = SUM( ConversionRate[spec] x SpecialtyPct[spec] / 100 )
Data Sources

Conversion rates are calibrated from Intuitive Surgical's published procedure mix data (2025 annual report: 3.1M procedures), AUA/ACOG/SAGES specialty adoption reports, and Cochrane systematic reviews on robotic vs. laparoscopic outcomes by procedure type.

5-Year Adoption Ramp Projection

Year-over-year robotic adoption follows a linear ramp starting at 40% of convertible cases in Year 1.

// Annual adoption ramp
AdoptionRate[yr] = min( 1.0, 0.40 + (yr - 1) x 0.15 )

// Gives: Y1=40%, Y2=55%, Y3=70%, Y4=85%, Y5=100%

ProjectedCases[yr] = ConvertibleLaparoscopic x AdoptionRate[yr]

// Design year = Year 3 (steady-state planning target)
DesignYearCases = ProjectedCases[3]
Why Year 3

Year 3 represents steady-state adoption after the learning curve plateaus. Sizing for Year 1 under-provisions; sizing for Year 5 over-provisions. P75-style planning: cover most scenarios without excess capital commitment.

Procedure Pareto & Gini Coefficient

Measuring Volume Concentration

Identifies which procedures drive the most volume and quantifies concentration using the Gini coefficient — borrowed from economics to answer: "Can we capture 80% of robotic opportunity by targeting just a few procedure types?"

ABC Classification Pareto

Procedures are sorted descending by volume. Cumulative percentage determines class assignment.

// Sort all procedures by annual case volume (descending)
// Compute cumulative volume percentage for each

Class A: cumulative volume ≤ 80% ← Automate first (highest ROI)
Class B: cumulative volume 80-95% ← Automate next
Class C: cumulative volume > 95% ← Low volume, defer

Gini Coefficient Concentration Metric

Computed via the trapezoidal rule on the Lorenz curve. A Gini of 0 = all procedures have equal volume. A Gini of 1 = one procedure has all volume.

// Build Lorenz curve from sorted procedure volumes
// x-axis: cumulative % of procedure types
// y-axis: cumulative % of case volume

// Trapezoidal integration
for each procedure i = 0..n:
  x0 = i / n,  x1 = (i+1) / n
  y0 = LorenzCurve[i-1],  y1 = LorenzCurve[i]
  AreaUnderLorenz += (x1 - x0) x (y0 + y1) / 2

// Gini = twice the area between the Lorenz curve and the diagonal
Gini = | 2 x AreaUnderLorenz - 1 |
What it tells the sales rep

Gini ≥ 0.7: Highly concentrated. A small set of procedures (e.g. prostatectomy + hysterectomy) drives most volume — ideal for targeted robotic programs. Gini < 0.4: Spread volume across many types — needs multi-specialty buy-in for ROI.

Seasonality & Demand Variability

How Predictable Is the Surgical Calendar?

Monthly Seasonality Model Demand Profiling

Each month is assigned a weight reflecting real surgical scheduling patterns, then scaled to the hospital's annual volume.

// Monthly weight factors (derived from surgical scheduling data)
Jan: 1.12  Feb: 1.08  Mar: 1.10  Apr: 1.02  May: 0.98  Jun: 0.92
Jul: 0.85  Aug: 0.88  Sep: 1.05  Oct: 1.08  Nov: 0.98  Dec: 0.74

MonthlyCases[m] = (AnnualCases x MonthWeight[m]) / SUM(AllWeights)
// + pseudorandom noise (seed-based, +/- 12%) for realistic variance

Coefficient of Variation (CoV) Stability Metric

Single number that tells the CFO whether volume is predictable (stable buy) or volatile (lease may be safer).

Mean = SUM(MonthlyCases) / 12
Variance = SUM( (MonthlyCases[m] - Mean)^2 ) / 12
StdDev = SQRT(Variance)

CoV = (StdDev / Mean) x 100

Class X: CoV < 15% = Stable ← Consistent utilization, buy with confidence
Class Y: CoV 15-30% = Seasonal ← Predictable peaks, plan staffing
Class Z: CoV > 30% = Erratic ← Volume swings, consider lease model

Design Day Analysis

How Many Cases Should the System Handle on a Busy Day?

Borrowed from warehouse engineering. Instead of sizing for the average day (under-provision) or the worst day (over-provision), we size for the P75 percentile — handles 75% of operating days without overflow.

Percentile Calculation Normal Approximation

Daily case volumes are modeled as a slightly overdispersed Poisson distribution, approximated with a normal distribution for percentile computation.

MeanDaily = AnnualCases / 250 ← 50 weeks x 5 operating days
StdDev = SQRT(MeanDaily) x 1.2 ← 1.2x overdispersion factor

// Standard normal z-scores for each percentile
P50: z = 0.000   P75: z = 0.674   P90: z = 1.282   P95: z = 1.645   P99: z = 2.326

Percentile[p] = max( 1, round( MeanDaily + z[p] x StdDev ) )

// System sizing recommendation
DesignDay = P75 ← Covers 75% of operating days comfortably
Why P75 and not P90 or P50

P50 (median) means the system is overwhelmed every other day. P90/P95 means paying for capacity used only 10-5% of the time. P75 is the capital-efficiency sweet spot: handles 3 out of every 4 operating days; overflow days absorb into scheduling flexibility.

System Recommendation Scoring

Multi-Factor Fit Score: 0-100 Points

The final recommendation is a composite score across five weighted dimensions. Not a black box — every point is traceable to a specific input.

Scoring Breakdown 100-Point Scale

// DIMENSION 1: Volume Fit (0-30 pts)
if DesignCases ≥ IdealVolume[sys]:   +30
elif DesignCases ≥ MinVolume[sys]:   +15 + 15 x (Cases - Min) / (Ideal - Min)
else:   +5 + warning flag

// DIMENSION 2: Specialty Coverage (0-25 pts)
CoverageRatio = MatchedSpecialties / HospitalSpecialties
SpecialtyScore = CoverageRatio x 25

// DIMENSION 3: Budget Fit (0-20 pts)
if Budget ≥ AvgSystemPrice:   +20
elif Lease/Usage model:   +15
else:   +5

// DIMENSION 4: Infrastructure Fit (0-15 pts)
Start at 15. Deduct -10 if OR sqft < minimum. Deduct -5 if ceiling < minimum.

// DIMENSION 5: Hospital Type Bonus (0-10 pts)
dV5 + Academic/400+ beds:   +10
X + Community/Rural:   +10
Xi (any hospital):   +7 ← proven workhorse bonus
Other combinations:   +3

// Final
FitScore[sys] = VolumeFit + SpecialtyCoverage + BudgetFit + InfraFit + TypeBonus
Recommendation = system with highest FitScore

ROI, Breakeven, and Total Cost of Ownership

Per-Procedure Economics

// Revenue per case
Robotic: $18,000
Laparoscopic: $12,000

// Cost per case
Robotic: $8,500
Laparoscopic: $6,000

IncrementalMargin
= ($18K - $8.5K) - ($12K - $6K)
= $3,500/case

5-Year TCO Components

SystemAcq = AvgPrice x SystemsNeeded
Instruments = Cases x CostPerCase x 5
Service = ServiceAnnual x 5
Training = NewSurgeons x $35K
Renovation = (if needed)

TCO = SystemAcq + Instruments
    + Service + Training + Renovation

ROI & Breakeven Calculation CFO Metrics

// Annual benefit components
IncrementalRevenue = NewCases x $15,000
LOS_Savings = DesignCases x (AvgLOS x 0.15) x $2,500/day ← 15% reduction
Complication_Savings = DesignCases x (CompRate x 0.20) x $25,000 ← 20% reduction
Marketing_Revenue = DesignCases x 0.10 x $15,000 ← 10% volume uplift

TotalAnnualBenefit = IncrementalRevenue + LOS + Complications + Marketing
AnnualNetBenefit = TotalAnnualBenefit - AnnualOperatingCost

// Payback period
PaybackMonths = (CapitalCost / AnnualNetBenefit) x 12

// 5-year ROI
FiveYearROI = ( (AnnualNetBenefit x 5) - CapitalCost ) / CapitalCost x 100%
Payer mix adjustment

Revenue is adjusted for the hospital's actual payer mix: Medicare (0.75x), Medicaid (0.55x), Commercial (1.15x), Self-pay (0.90x). A hospital with 60% Medicare sees lower effective reimbursement than one with 60% commercial. The weighted reimbursement factor produces a hospital-specific revenue projection, not an industry average.

Utilization & Capacity

How Many Systems, and Will They Stay Busy?

System Capacity Model Operations

// Weighted average case duration (by specialty mix)
Urology: 2.5h  Gyn: 1.5h  General: 1.5h  Thoracic: 3.0h  Colorectal: 3.5h
AvgDuration = SUM( Duration[spec] x Pct[spec] )

SetupTurnover = 0.75 hours ← draping, docking, undocking per case
TotalPerCase = AvgDuration + SetupTurnover

AvailableHours = 10 hours/day ← standard OR block time
MaxCasesPerDay = FLOOR( AvailableHours / TotalPerCase )

SystemsNeeded = CEIL( DesignCases / (MaxCasesPerYear x 0.80) )
// Target 80% utilization — leaves room for maintenance, emergencies

UtilizationPct = DesignCases / (Systems x MaxCasesPerYear) x 100
