1. Context

This report presents the direct-cost structure to build and operate Torna Idioma — a university-grade, speaking-first AI program that teaches Filipinos to speak Spanish (Método Rizal). It is delivered under Digit2AI's AI-Native model: a multi-agent engineering workforce generates and iteratively tests the scaffolding, compressing the build to ~1,350 effective engineering hours while preserving enterprise quality (JWT auth, PostgreSQL, additive migrations, rate limiting, 99.9% target availability).

A distinctive cost advantage: the speaking experience runs on free voice infrastructure — Microsoft Edge neural TTS (speech output) and the browser's speech recognition (speech input) — so recurring AI spend is essentially Claude-only, a fraction of typical voice-AI products. All figures reflect direct cost with no margin or VAT.

2. Technical Architecture

Frontend — React 18 + Vite single-page app; shared EN/FIL i18n; installable PWA (manifest + service worker + icons); built to static dist and served from the edge.
Backend — Node.js + Express vertical mounted at /Torna_Idioma; JWT auth; modular routes: speaking, vocab/SRS, placement, emperador, rizal, tutor/TTS; per-user rate limiting.
AI layer — Anthropic Claude (tutor, spoken-attempt feedback, 5-criterion oral rubric, placement); Microsoft Edge neural TTS (free) with ⟦es⟧ smart-split voice; browser speech recognition (free); SM-2 spaced-repetition engine; (future) Azure per-phoneme Pronunciation Assessment.
Data — PostgreSQL (Render); multi-tenant ti_* tables; additive, reversible migrations.
Infrastructure — Render web service with auto-deploy on push to main; custom domain tornaidioma.com + SSL; static/CDN delivery.
Content — CEFR A1→B2 curriculum (12 modules, 425 root words / ~1,200–1,700 word forms), 6 A1 speaking units, TESDA evidence guides, adaptive placement bank, QA gates G1–G5.
