1. Contexto y Hallazgos de Ingeniería Inversa

Este informe presenta la estructura de costo directo para construir y operar una réplica funcional de Planea — un copiloto financiero personal para el mercado colombiano que entrega un diagnóstico financiero gratuito en 60 segundos, un puntaje de salud financiera de 0–100 y un asistente de IA conversacional ("Maya") que guía al usuario semana a semana. La réplica se entrega bajo el modelo AI-Native de Digit2AI: una fuerza de trabajo de agentes de ingeniería genera y prueba iterativamente el andamiaje, comprimiendo la construcción a ~1.550 horas efectivas de ingeniería conservando calidad empresarial (autenticación, aislamiento de datos por usuario con RLS, migraciones reversibles, cifrado, disponibilidad objetivo 99.9%).

El alcance y la pila tecnológica se derivaron por ingeniería inversa del producto en vivo (superficie pública y bundle del cliente, sin acceder a datos privados). Hallazgos clave del stack original:

  • Frontend: React + Vite empaquetado con Ionic + Capacitor — una sola base de código que se publica como PWA web y como apps nativas iOS/Android.
  • Backend / BaaS: Supabase (PostgreSQL gestionado + autenticación + Edge Functions + almacenamiento), con el aislamiento entre usuarios apoyado en políticas de Row-Level Security.
  • Capa de IA ("Maya"): el proveedor del modelo de lenguaje se ejecuta del lado del servidor (Edge Function) — sin exposición de llaves en el navegador. Buena higiene de seguridad.
  • Datos financieros: marco de Open Finance / banca abierta de Colombia, acceso de solo lectura, sin ejecutar transacciones.
  • Infraestructura: hosting en Vercel; analítica con Segment + Google Analytics.
  • Metodología: puntaje financiero ponderado sobre 4 pilares (fondo de emergencia, flujo de caja, salud de deudas, estabilidad de ingresos) que se expande a 7 áreas de planeación, alineado a la metodología CFP Board® adaptada al mercado colombiano.

Todas las cifras de este documento reflejan únicamente el costo directo de entrega y operación, sin margen de utilidad ni IVA.

2. Arquitectura Técnica de la Réplica

Frontend — React 18 + Vite sobre Ionic + Capacitor: una base de código → PWA instalable + apps nativas iOS/Android. Tema claro/oscuro, i18n (es-CO), navegación móvil-primero.
Backend / BaaS — Supabase: PostgreSQL gestionado, Auth (email/contraseña + recuperación), Edge Functions (lógica de Maya, scoring y conectores), Storage. Sin servidor propio que mantener.
Capa de IA — Maya — Anthropic Claude tras Edge Function (llave del lado servidor), con RAG sobre los datos financieros del usuario, guía semanal y respuestas a preguntas; rúbrica de recomendaciones por las 7 áreas de planeación.
Motor de Scoring — cálculo determinístico 0–100 ponderado por los 4 pilares, parametrizado al mercado colombiano (metodología CFP adaptada); histórico de puntaje versionado.
Open Finance — conector a agregador de banca abierta (p. ej. Belvo / Finerio), solo lectura, normalización de movimientos, consentimiento explícito y revocable.
Datos — PostgreSQL multi-usuario con Row-Level Security estricta; migraciones aditivas y reversibles; cifrado en reposo y en tránsito.
Privacidad y cumplimiento — Ley 1581 (habeas data), acceso de solo lectura, borrado de datos a solicitud, registro de consentimiento; la app nunca ejecuta transacciones.
Infraestructura — Vercel (frontend + edge) con auto-deploy; Supabase gestionado; dominio + SSL; analítica Segment + Google Analytics; observabilidad con Sentry.
