1. Overview

The Load-to-Load Matching Engine automatically scans the active load board to find pairs of shipments that can be combined into a single truck movement. This reduces empty miles, increases carrier utilization, and improves margin per load.

3.2 Timing Score (30% weight)

Evaluates whether the truck can complete Load A's delivery and reach Load B's pickup within the time window.

Gap Between LoadsScoreFeasibility

Note on Date-Only Data: When load data contains only dates (no timestamps), the algorithm assumes a 12-hour delivery window. A 1-day gap between loads computes as ~24 hours. For optimal matching, pickup/delivery windows with time-of-day produce significantly better results.

4. Hard Filters & Thresholds

Before scoring, the algorithm applies hard filters to eliminate non-viable candidates:

Max Candidates200 per searchPerformance limit
Max Deadhead150 milesMaximum empty miles between loads
Max Time Gap48 hoursMaximum wait between delivery and next pickup
Minimum Score30 (composite)Pairs below this are not returned
Composite Score Formula:
Score = (Route × 0.35) + (Timing × 0.30) + (Equipment × 0.20) + (Revenue × 0.15)

A pair must score ≥ 30 to be returned in results. Pairs scoring ≥ 70 are flagged as high-confidence matches.

5. Data Flow

  1. User enters Load ID (load reference number from TMS)
  2. System finds anchor load in the database (by ID or reference number)
  3. Hard filters applied — select up to 200 candidates matching equipment, date window, and status
  4. Each candidate scored across 4 dimensions
  5. Results ranked by composite score, top 20 returned
  6. User accepts or rejects pairs — feedback stored for algorithm improvement
  7. Accepted pairs saved to lg_load_pairs table for dispatch

7. Recommendations for Optimal Results

Data Quality Drives Match Quality. The algorithm performs best when load data includes:
  1. Pickup/delivery time windows (not just dates) — enables precise timing scores
  2. Mileage data per load — enables accurate RPM and deadhead calculations
  3. Sell rates alongside buy rates — enables margin optimization scoring
  4. Structured city + state (separate columns) — enables precise geographic matching
  5. Equipment type standardization (dry_van, reefer, flatbed) — prevents false mismatches
  6. Load volume ≥ 500 active loads — larger pools produce more viable pairs

Tunable Parameters

All thresholds are configurable per client. Common adjustments include:

  • Increasing max deadhead to 200-250 miles for rural/sparse markets
  • Extending time gap to 72 hours for less time-sensitive freight
  • Lowering minimum score to 25 to surface more marginal pairs for review
  • Adding preferred lane weighting to prioritize known high-volume corridors
