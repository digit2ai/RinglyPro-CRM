03 · Alcance del MVP

Qué se construye en esta primera versión.

El alcance se concentra en una versión comercial funcional y enfocada. La arquitectura detallada, el modelo de datos, las APIs, los prompts, las políticas RLS y el checklist técnico quedan en el anexo técnico.

Incluye

  • Fundación técnica sobre Next.js + Vercel + Supabase
  • Autenticación y perfil financiero
  • Puntaje Planea 2.0 (versión básica explicable)
  • Asistente con IA y contexto del usuario
  • Plan financiero simple y accionable
  • Dashboard inicial con próxima mejor acción
  • Row-Level Security y seguridad base
  • Carga de PDF/CSV/XLSX
  • Análisis documental básico
  • Panel administrativo operativo básico
  • QA funcional, smoke tests y documentación

No incluye (en esta etapa)

  • Open banking o conexión bancaria directa
  • Movimiento de dinero o ejecución de pagos
  • Asesoría de inversión o recomendación regulada
  • App nativa publicada en tiendas
  • OCR avanzado de imágenes complejas
  • Marketplace financiero
  • Integraciones con burós de crédito
  • Soporte 24/7
  • Cumplimiento legal/regulatorio especializado

04 · Arquitectura AI-First

Una base moderna, segura y lista para escalar — sin sobre-ingeniería.

El MVP se construye sobre una pila deliberadamente simple y económica, que ya es estándar en productos financieros de alto crecimiento. Cada capa está pensada para validar rápido hoy y crecer mañana sin reescribir.

Frontend
Aplicación Next.js sobre Vercel: web responsiva, instalable, móvil-primero, con tema claro/oscuro y experiencia de chat fluida con respuestas en streaming.
Datos y Auth
Supabase: PostgreSQL gestionado, autenticación, almacenamiento y Row-Level Security estricta — cada usuario solo ve sus propios datos. Migraciones aditivas y reversibles.
Asistente IA
Modelo de lenguaje líder tras una Edge Function (llaves siempre del lado servidor), con contexto del perfil financiero del usuario, respuestas guiadas y barandas — explica, no ejecuta transacciones.
Motor de Puntaje
Cálculo determinístico y explicable por pilares, parametrizado al mercado local; histórico y tendencia del puntaje para mostrar progreso.
Documentos
Carga de PDF/CSV/XLSX con análisis básico que enriquece el perfil del usuario y alimenta la conversación con el asistente.
Operación y QA
CI/CD con despliegue continuo, panel administrativo, analítica de producto, observabilidad, smoke tests y verificación funcional antes de cada entrega.

05 · Caso de negocio

Qué decisión podrá tomar Planea después de este MVP.

Esta etapa debe evaluarse como una prueba rápida y de bajo riesgo para saber si Planea puede convertir sus avances actuales en un producto financiero AI-First con tracción comercial real.

Validación 01

¿El usuario entiende mejor su situación?

El dashboard y el Puntaje Planea permiten medir si el usuario comprende su estado financiero y sus prioridades.

Validación 02

¿El usuario conversa con Planea?

El asistente mide preguntas reales, objeciones, intención, necesidades frecuentes y calidad percibida.

Validación 03

¿El plan financiero genera acción?

El plan simple revela si el usuario acepta recomendaciones, guarda metas o vuelve a revisar su progreso.

Validación 04

¿Los archivos agregan valor?

La carga inicial valida si los usuarios están dispuestos a enriquecer su perfil con documentos financieros.

06 · Cronograma ejecutivo

Un MVP de 4 semanas, con una quinta de estabilización.

Cada semana cierra con algo funcional y demostrable. No hay "fase de integración" sorpresa al final: lo que se construye, se prueba y se muestra.

Semana
1

Foundation + Auth + Perfil financiero

Proyecto Next.js, Vercel y Supabase; autenticación, RLS base, layout y perfil financiero mínimo.

Semana
2

Puntaje Planea 2.0 + Dashboard inicial

Score básico, resultado visual, explicación simple, dashboard autenticado y próxima mejor acción.

Semana
3

Asistente con contexto + Plan financiero simple

Asistente con respuestas en streaming, contexto del usuario, gateway de IA, respuestas guiadas y generación del plan financiero simple.

Semana
4

Uploads + Análisis documental + Admin + QA

Carga PDF/CSV/XLSX, análisis básico, panel administrativo, smoke tests, validación funcional y documentación.

Semana
5

Buffer opcional de estabilización

Correcciones, ajustes menores, hardening y preparación final para una producción controlada.

Entrega final: MVP funcional en ambiente de staging o producción controlada, con los flujos críticos validados y documentación técnica breve para continuidad.
