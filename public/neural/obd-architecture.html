TREATMENT EXECUTOR — INTERNAL ARCHITECTURE

TRIGGER LAYER

executor.trigger(clientId, event, data)

1. Look up active treatments matching event
2. Rate limit: 1 execution per phone per hour
3. Fetch client config (API keys, phones)
4. Pass to execution layer

EXECUTION LAYER

_executeTreatment(treatment, data, config)

1. Loop through actions array
2. Handle delays (immediate / scheduled / pending)
3. Template variable substitution
4. Execute via action handler

LOGGING LAYER

treatment_execution_log table

1. Log every trigger attempt
2. Record actions executed + results
3. Track status (completed / partial / failed)
4. Feed ROI dashboard calculations
