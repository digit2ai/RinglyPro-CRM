Demo script, objection handling, and email reply

1
Hospital Intake
Creates the hospital profile that everything builds from
▼
How the system does it
  • Manager types a hospital name into a single text field
  • System creates a project record in the database with a unique ID
  • That project ID links every analysis, recommendation, survey, business plan, and tracking record together
  • No other input needed -- the AI takes over from here
How to show Greg
  • Type the hospital name
  • Click create
  • Move on immediately -- keep it fast
"Greg, this is where your Manager starts. They type in a hospital name. That's it. One input."
2
Analysis
AI researches the hospital -- replaces days of manual work
▼
How the system does it
  • AI agent sends the hospital name to Claude with a structured research prompt
  • Claude searches its training data for bed count, surgical volumes, specialties, robotic program history, and market positioning
  • System queries CMS Hospital Compare API for that specific hospital -- pulls readmission rates, mortality rates, infection rates, and length of stay data
  • System ingests the hospital's annual report (PDF upload or URL) and extracts procedure counts, revenue data, robotic surgery stats, and surgeon counts using AI document analysis
  • System calculates: total surgical volume, approach breakdown by specialty (% open, % laparoscopic, % robotic), procedure pareto, monthly seasonality, weekday OR distribution, hourly utilization, surgeon capacity modeling, and competitive landscape
  • All results stored as structured JSON linked to the project ID
How to show Greg
  • Click run analysis
  • Walk through results: procedure volumes, approach mix percentages, CMS outcomes
"This is the hospital's own published data. The AI pulled it in seconds. Your Manager didn't Google anything, didn't read an annual report, didn't search CMS. It's done."
3
System Match
AI recommends which da Vinci system fits this hospital
▼
How the system does it
  • Loads the complete da Vinci catalog with specs for each model (Xi, Xi Dual Console, dV5, dV5 Dual Console, SP, X)
  • Pulls the hospital's analysis data: surgical volume, specialty mix, OR count, growth trajectory, budget range
  • Runs a weighted scoring algorithm: volume capacity, specialty coverage, infrastructure fit, growth headroom, budget alignment, competitive positioning
  • Produces a ranked recommendation with scores and reasoning for each system
How to show Greg
  • Show the recommendation with scoring
"Based on the data, the AI recommends the Xi for this hospital. Here's why. Your Manager didn't have to figure this out -- the data made the decision."
4
Presentation
AI generates a slide deck with voice narration

  • AI voice narration scripts generated per slide

How to show Greg

"Your Manager walks into the CEO's office with this ready. No PowerPoint. No formatting. This took zero hours to build. It used to take how long?"
5
Surgeon Surveys
Replaces Survey Monkey -- emails surgeons, collects commitments
▼
How the system does it
  • Manager creates a survey -- system type and hospital name pre-filled from previous steps
  • Default questions auto-loaded: incremental volumes, procedure breakdown, barriers, competitive leakage, formal commitment
  • Manager adds surgeon recipients -- name, email, specialty (single or CSV bulk upload)
  • Each recipient gets a unique personal token stored in the database
  • Manager clicks "Send via Email" -- system calls SendGrid API for each recipient
  • Surgeon receives a branded HTML email with a personal link to a secure web form
  • Survey form asks: How many more procedures per month? What procedure types and breakdown? What prevents more cases? How many cases going to competitors? Will you formally commit?
  • Surgeon fills it out in 3 minutes -- no login required
  • Response stored in the database: surgeon name, specialty, monthly cases, procedure breakdown, barriers, competitive leakage, commitment status
  • Manager sees all responses with completion tracking
How to show Greg
  • Create a survey
  • Add 2-3 surgeon recipients
  • Click "Send via Email"
  • Show the email that goes out
  • Open the public survey link to show what the surgeon sees
"The surgeon gets this email, clicks the link, fills out the form. When they submit, their numbers are sitting right here. No Survey Monkey. No manual compilation. No re-typing into a spreadsheet."
6
Business Plan
Two ROI layers: surgeon volume revenue + clinical outcome savings
▼
How the system does it -- Surgeon Volume ROI
  • Manager clicks "Import from Survey" -- all surgeon responses convert into commitments in the business plan
  • Monthly case count split across procedure types using the surgeon's percentage breakdown
  • System queries DRG reimbursement database for each procedure type -- returns average blended rate (Medicare + commercial payer mix)
  • Annual cases = monthly x 12
  • Revenue per surgeon = annual cases x DRG rate per procedure
  • Surgeon who said "yes" to commitment gets status "confirmed"
  • Total incremental revenue summed across all surgeons and procedures
  • Manual entry also available for field conversations
How the system does it -- Clinical Outcome ROI
  • System pulls volume_projection from Step 2 analysis -- annual cases by specialty, % open, % lap, % robotic
  • Case data auto-populates the dollarization table (no manual entry)
  • Manager clicks "Calculate Savings"
  • Engine loads clinical evidence library -- peer-reviewed studies by specialty
  • For each specialty calculates: LOS savings (days reduced x cost/day), complication savings (rate reduction x cost/complication), readmission savings (rate reduction x cost/readmission), transfusion savings (rate reduction x cost/unit)
  • Results include citations to published studies
How the system does it -- Combined Totals
  • Manager clicks "Recalculate Totals"
  • Incremental Cases/Year = sum of all surgeon commitments
  • Incremental Revenue = sum of all DRG-calculated revenue
  • Clinical Outcome Savings = sum of all dollarization savings
  • Combined ROI = revenue + clinical savings
  • Payback Period = (system price x quantity) / annual net benefit x 12 months
  • Five-Year Net Benefit = (annual net benefit x 5) minus system cost
How to show Greg
  • Click "Import from Survey" -- show surgeon commitments populate with revenue
  • Show clinical dollarization auto-populated from analysis
  • Click "Calculate Savings" -- show results by specialty
  • Click "Recalculate Totals" -- show combined ROI
"The surgeon said 8 cases a month of radical prostatectomy. The system looked up the DRG rate, multiplied by 12, and that's the annual revenue. No spreadsheet."
"This is the hospital's own CMS data cross-referenced with published clinical evidence. We're using their own numbers. With citations."

7
Plan Tracking
Compares actual surgeon performance vs. business plan projections
▼
How the system does it
  • Every da Vinci robot logs case data: robot serial, surgeon name, procedure type, date, console time, instruments used
  • Intuitive's internal systems push data to the SurgicalMind Robot Data API (POST endpoint, batch case records, API key auth)
  • Each case stored with unique constraint to prevent duplicates
  • Manager clicks "Sync Robot Data" and selects a time period
  • System queries all robot cases for that hospital and date range
  • Fuzzy-matches each case to committed surgeons in the business plan
  • Calculates projected cases for period: annual commitment prorated to months in period
  • Calculates variance: actual minus projected (absolute and percentage)
  • Creates actuals record with per-surgeon breakdown
  • Dashboard shows: timeline chart (projected vs actual over time), per-surgeon table, summary cards, color coding (green/yellow/red), ROI realization %
  • Executive summary generates text report for customer follow-up
  • Snapshot feature saves point-in-time records for historical comparison
How to show Greg
  • Show the tracking dashboard
  • Show "Sync from da Vinci Robot"
  • Show variance tracking and per-surgeon breakdown
"Your robots already log every case. That data feeds directly into this dashboard. It matches each case to the surgeon's original commitment."
"Six months after the sale, your Manager goes back to the CFO and says: here's exactly how the investment is performing. Dr. Smith committed to 96 cases a year, he's done 84 so far, he's on track. That's the conversation that sells the next system."
